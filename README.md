# obsidian-claude-plugin

Obsidian vault integration for Claude Code. Auto-save sessions, capture git commit context, create and retrieve notes, and open notes in the Obsidian GUI.

## Features

- **First-run setup** — `/obsidian:setup` wizard configures your vault path, domain taxonomy, and routing rules
- **Save conversations** — export Claude sessions to the right project folder automatically
- **Find notes** — search your vault by keyword, tag, or topic
- **Create notes** — new notes or full project folder structures
- **Daily note** — append to or read today's daily note
- **Reorganize** — deep vault analysis with a dedicated subagent
- **Session-end hook** — auto-saves significant sessions via background `claude --print` summarizer
- **Commit capture** — appends goal/decision context to a vault note after every git commit
- **Obsidian URI integration** — opens notes directly in the GUI after writing (macOS and Windows/WSL)

## Commands

| Command | Usage |
|---|---|
| `/obsidian:setup` | First-run wizard — configure vault path, domains, and routing |
| `/obsidian:save [topic]` | Save current session |
| `/obsidian:find <query>` | Search vault |
| `/obsidian:new <title>` | Create note or project |
| `/obsidian:daily` | Open/append today's daily note |
| `/obsidian:bookmark` | Drop a chapter marker in the session |
| `/obsidian:recall <topic>` | Cross-search vault, git log, and GitHub PRs |

## Installation

```bash
claude plugin install nhangen/obsidian
```

Then run the setup wizard:

```
/obsidian:setup
```

The wizard prompts for your vault path, domain names, keywords per domain, and daily/inbox paths, then writes `obsidian.local.md`.

## Config

`obsidian.local.md` is machine-specific and gitignored. It is generated by `/obsidian:setup` or you can copy and edit the example manually:

```bash
cp obsidian.local.md.example obsidian.local.md
```

Key fields:

```yaml
vault_path: /absolute/path/to/your/vault
vault_name: Obsidian          # must match vault name in Obsidian app
daily_path: Daily/            # relative to vault root
auto_save: true               # set false to disable session autosave
auto_open: true               # open note in GUI after writing
```

Routing is driven by the `## Routing Rules` section — plain English keyword lists that Claude uses to decide which folder a note belongs in. `/obsidian:setup` generates these from your answers; edit them directly to tune routing.

## Vault structure

The plugin works with any vault structure. `/obsidian:setup` generates paths based on your domain names. Example output for domains `Work, Personal, Development`:

```
vault/
├── Projects/
│   ├── Work/
│   ├── Personal/
│   └── Development/
├── Daily/
└── Inbox/
```

Sessions are routed by keyword detection against your routing rules. Ambiguous content falls back to `Inbox/`.
